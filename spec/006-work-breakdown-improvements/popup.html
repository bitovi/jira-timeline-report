Work Breakdown — Epic popupDetail popup

Clicking a child issue (an Epic row/cell) in the work breakdown opens a small popup with its rollup and per‑work‑type dates. Today (src/canjs/controls/issue-tooltip.js) that popup is unstyled: a bare title with a "❌" text glyph for close, plain "Show Children" text link, colored label pills for rollup / dev / qa / uat each followed by wordy "Start: … / End: …" paragraphs, and warnings rendered as a plain colored `<div>`. The mock below keeps all the same underlying data (target dates, "was" dates, warning messages, and the driving issue links for start/due) but restyles it with the same visual language as the other work‑breakdown mocks: a colored header strip for overall status, compact status pills, and work‑type swatch badges (matching the matrix's d/D/Q/U letters) with the start/due driving-issue detail shown inline instead of as verbose paragraphs.

What changed

  • Header is colored by overall rollup status (like the card headers elsewhere) instead of plain text.
  • Close is a real button (✕ in a rounded hit-target) instead of a bare "❌" glyph in the text flow.
  • The overall rollup status and date range live once, in the subheader — the old separate "Rollup" row duplicated that same status/date and has been removed.
  • Each work type gets a swatch letter (D/Q/U) matching the matrix mock's d/D/Q/U keys, so the popup and the board read as one visual language.
  • Status is spelled out as a pill/word (Behind, Complete) next to the dates, not just color.
  • A slipped date shows its previous value inline (was Jun 19) instead of a trailing parenthetical.
  • The "Start: … / End: …" paragraphs become a compact, always-visible two-column strip (start issue/reason, due issue/reason) instead of full sentences.
  • Warnings get an alert-styled banner (border + tint) instead of an unstyled colored block.
  • "Show Children" becomes a single, clearly-labeled "Explore children →" footer action (no duplicate link).

Popup vs. modal

Recommendation: keep it a popup, not a modal. The work breakdown is a scanning workflow — you're comparing several cards' statuses at once, and an anchored popup lets the rest of the board stay visible and in place while you check one epic's detail, then click another card without an extra close step. A centered modal with a dimming backdrop hides that context and adds friction for what's usually a quick "why is this behind?" glance. The content here (one rollup row + up to 4 work types) is also small enough that an anchored popup never needs its own scroll. A modal would make sense if this grew into a much bigger surface (e.g. a full child‑issue table) — but that's exactly what the "Explore children →" action is for: it navigates to a dedicated, full‑width view instead of cramming everything into the popup.
