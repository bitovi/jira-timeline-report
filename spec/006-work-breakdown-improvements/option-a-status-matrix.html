Work Breakdown — Option AStatus matrix

Each card's children become a compact matrix: rows are child issues, columns are work types. The work‑type letters (d D Q U) and their rollup target dates appear once in the header (a red date below means it slipped from a previous period; a teal date means it moved forward — pulled earlier); each row is then just a tight grid of rounded status swatches — same shape as the header keys — so a glance down the Q column answers "who's behind on QA?" while fitting in narrow horizontal space. Use the Big / Tight toggle (like the current work breakdown) to trade legibility for density. Cells distinguish three states, matching the Gantt: a colored swatch (work with dates), a tan ∅ swatch (work exists but has no dates — the app's "unknown" status), and a blank outline (no work of that type for this issue). The Secondary report toggle switches between Status (a single rollup-status column + Target Delivery date per child) and Work breakdown (the full four-work-type matrix) — the same card, one column vs. four.
