Work Breakdown — Option BProgress rollup

Keeps the familiar card + child‑list shape but adds magnitude: each work type shows a progress bar, a done / total count, and its target date (a red date in parentheses means it slipped from a previous period). Each child gets its own mini meter and %. Answers "how close are we?" — not just "what color is it?".

Work types: design / dev / qa / uat  ·  bar fill = % of that work type's children complete; child meter = that issue's overall completion.
