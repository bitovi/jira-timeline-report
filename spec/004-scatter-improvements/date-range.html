Scatter Plot — start / end date range filter

Design exploration for letting users constrain the Scatter Plot to a start / end date window. The primary home is the existing Filters menu (Option A). Alternatives — View Settings, an inline controls-bar widget, and an on-axis drag-select — are shown for comparison. See date-range.md for the implementation plan.

Option A — Filters menu Recommended

A new “Due date range” row inside the existing Filters dropdown, matching the two-column FilterGrid layout used by the Status and Issue-type filters. Two Atlaskit date pickers (From / To), a Clear action, and a short row of quick presets. Empty fields mean “unbounded”, so it degrades to today’s behavior.

Pros

  • Matches the request; lives with the other issue filters.
  • Reuses the FilterGrid / DropdownMenu patterns already in place.
  • Presets + explicit pickers cover both quick and precise needs.

Cons

  • Adds height to an already-tall menu.
  • Filtering is hidden behind a click (not visible at rest).

Filter vs. clamp: with filter the greyed issues are removed and the axis re-fits to Q1. With clamp (a possible follow-up) the same issues stay in the data but the axis is pinned to Jan–Mar and off-window markers scroll out of view. See Question #1 in the plan.

Recommendation

Ship Option A (Filters menu, two date pickers + Clear) with Option E preset chips inside the same section. Use filter semantics first (remove out-of-window issues, let the axis re-fit). Consider Option D (axis drag-select) and a “clamp axis” toggle as follow-ups. Scope the new section to the scatter ('due') report so it doesn’t leak into other report types.
