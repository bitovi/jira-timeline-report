Scatter Plot — UI improvement ideas

Static HTML/CSS explorations for the scatter timeline report, styled to match the real component (src/react/reports/ScatterTimeline/) and Atlassian design tokens. Everything here is pure HTML/CSS — no charting library. Each section renders the same fixture data as the live report so differences are apples to apples. This is for reporting, so ideas favor legibility, print-friendliness, and self-explanatory output over interaction.

Currentas-is

Faithful replica of today's report. Note the two annotated problems: the gap above the first row of pills is much smaller than the gap below the last row, and the plain gray pills sit visually detached from their status dots.

Things worth fixing (beyond the spacing you spotted):
  • Vertical rhythm: markers are pinned at top: 4px in a 40px row, so there's ~4px above the first pill but ~16px below the last — the asymmetry you noticed. Centering markers in the row fixes it.
  • Detached dot & pill: the gray pill and the colored dot read as two objects with a gap between them, so the status color feels like an afterthought rather than part of the item.
  • No legend: nothing on the page says what green / blue / yellow / orange mean. Critical for a report that gets screenshotted or exported.
  • Yellow vs. orange: behind (#f5cd47) and warning (#ff8e09) are easy to confuse at dot size, especially in print or on projectors.
  • Truncation with no recourse: long labels get cut with an ellipsis and only a native title tooltip — useless in a static export. Consider wider pills / wrapping for reports.
  • Busy grid: every month draws a full-height vertical line, competing with the data. Atlassian charts keep gridlines light and emphasize larger units (quarters).
  • Bare today line: a 1px orange line with no label; readers can't tell it's "today" without context.

Idea 1 — Balanced vertical rhythmrecommended

Center each marker within its row instead of pinning it to top: 4px. Equal breathing room above and below every pill, and the top/bottom of the whole plot now match. Tiny change, immediately looks more deliberate.

Idea 2 — Cohesive status chipsrecommended

Keep today's compact layout — the status dot still sits on the pill's right edge, so chips pack just as tightly — but swap the gray fill for a white pill with a 1px hairline border. Reads cleanly on white and in print, and looks more like an Atlassian surface than a flat gray box.

Idea 3 — Click to open the issuerecommended

Each chip becomes a real link to the Jira issue. Hover / keyboard focus lifts the chip, shows a focus ring, and reveals an ↗ open-in-Jira glyph. Fully keyboard accessible. Hover or tab through the chips below.

Releases (which have no issue URL) would simply render as non-links.

Idea 4 — Report header & status legendrecommended

A compact header gives the report a title, date range, and a status legend with counts — so an exported image is self-explanatory. Legend chips reuse the exact status colors.

Idea 5 — Lighter grid, emphasized quarters & a "Today" flag

De-emphasize month lines, draw crisp lines only at quarter boundaries, and add a subtle alternating tint per quarter to help the eye group time. The today line becomes a labeled dashed marker. Also adds gentle zebra striping for row scanning.

Idea 6 — Everything together

The recommended set combined: header + legend, balanced rhythm, cohesive clickable chips, lighter quarter-emphasized grid, and a labeled today flag.
